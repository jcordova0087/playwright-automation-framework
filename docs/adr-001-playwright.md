# ADR-001 - Why Playwright?

## Status

Accepted

---

## Context

The framework requires a modern automation tool capable of supporting UI and API testing while providing excellent developer experience.

---

## Decision

Playwright has been selected as the primary automation framework.

---

## Reasons

- Cross-browser support
- Native parallel execution
- Built-in API testing
- Excellent TypeScript support
- Auto-waiting capabilities
- Powerful debugging tools
- Trace Viewer
- Screenshots and Videos
- Active community
- Frequent updates

---

## Consequences

Advantages:

- Faster test execution.
- Cleaner code.
- Better debugging.
- Easier maintenance.

Potential drawbacks:

- Smaller ecosystem compared to Selenium.
- Requires modern Node.js environments.

Overall, Playwright provides the best balance between productivity, maintainability, and scalability for this project.
